# akb

A CLI tool that analyzes source code repositories and generates structured markdown knowledge bases for AI agents.

AI coding assistants like Claude Code, GitHub Copilot, and Cursor work best when they have context about your codebase. akb walks your repository, sends each source file to Claude for analysis, and produces a set of markdown documents that describe your code — one per source file, mirroring your directory structure.

The generated markdown is intended to be used with tools like [qmd](https://github.com/tobi/qmd) to help agents gather context about a project.

**Note:** This is a prototype. It is not yet clear whether the generated knowledge base brings meaningful benefit over agents reading the source code directly.

## Installation

Requires Go 1.25+ and the [Claude CLI](https://docs.anthropic.com/en/docs/claude-cli).

```sh
go install github.com/esnunes/akb@latest
```

Verify the Claude CLI is installed:

```sh
claude --version
```

## Usage

akb has two subcommands: `init` and `generate`.

### Init

Discover file extensions in your repository and classify them as source code or non-source:

```sh
akb init --path /path/to/repo
```

This creates a configuration file at `docs/akb/.config.yaml`:

```yaml
# Generated by akb init. Edit as needed.
source_extensions:
  - .go
  - .py
  - .ts
exclude_patterns:
  - .git/
  - node_modules/
  - vendor/
  - docs/akb/
```

Edit this file to add or remove extensions and exclusion patterns before running `generate`.

### Generate

Analyze source files and generate markdown documentation:

```sh
akb generate --path /path/to/repo
```

Options:

- `--workers N` — number of concurrent Claude CLI calls (default 5, max 20)
- `--force` — regenerate all files, ignoring the cache
- `--verbose` — enable debug logging

Output is written to `docs/akb/`, mirroring the source tree:

```
docs/akb/
├── .config.yaml
├── .manifest.json
├── main.go.md
└── internal/
    ├── config/
    │   └── config.go.md
    └── walker/
        └── walker.go.md
```

On subsequent runs, only new or changed files are re-analyzed. Markdown for deleted source files is automatically cleaned up.

### Flags

Both subcommands accept:

- `--path <dir>` — repository path (default: current directory)
- `--verbose` — enable debug logging

## How It Works

1. **Init** walks the repository, collects unique file extensions, and asks Claude to classify which are source code. The result is saved as a YAML config.
2. **Generate** loads the config, walks source files matching the configured extensions, and computes SHA-256 hashes. Files whose hash hasn't changed since the last run are skipped. Changed files are sent to Claude for analysis using a worker pool, and the resulting markdown is written alongside a manifest for future cache lookups.
